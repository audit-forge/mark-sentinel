How it works

Three steps to a compliance-ready report.

1

Point it at your infrastructure

Run against a Docker container, a Kubernetes pod, or a direct connection. No agent installation, no network changes required.

2

Checks run automatically

Hundreds of security checks execute against your configuration, runtime state, and container settings — each mapped to a named control in CIS, NIST, or DISA STIG.

3

Get findings your tools already understand

SARIF 2.1.0 for GitHub Code Scanning. Evidence bundles for auditors. Wiz and SCC JSON for cloud posture platforms. CSV for spreadsheet review.

Output

Structured findings, not log noise.

Every finding includes the control ID, what was found, what was expected, a remediation step, and the evidence that produced it.

# Redis — direct connection audit python3 audit.py \ --mode direct \ --host 10.0.0.5 \ --port 6379 \ --sarif out.sarif \ --bundle evidence.zip \ --csv findings.csv [FAIL] RD-AUTH-001 CRITICAL Default user must not have nopass Actual: user default on nopass ~* +@all Expected: password-protected default user Fix: Set requirepass or configure ACL [FAIL] RD-CFG-002 HIGH Protected mode must be enabled Actual: protected-mode no Expected: protected-mode yes [WARN] RD-CFG-008 MEDIUM Syslog integration recommended

SARIF 2.1.0

Upload directly to GitHub Code Scanning. Rules deduplicated per check ID, tagged with NIST controls and FedRAMP mappings.

Evidence bundle

ZIP archive with results JSON, SARIF, runtime snapshot, and per-check evidence files. Drop it in your GRC tool or hand it to an auditor.

Wiz & Google SCC

Push findings directly to your cloud security posture management platform via Wiz Issues API or GCP SCC Findings API v1.

Tools

What's covered.

redis-stig-audit v1.0

CIS Redis Benchmark checks for protected mode, TLS, ACL configuration, persistence, and container hardening. Docker, Kubernetes, and direct modes.

37 tests CIS Redis FedRAMP CMMC
elastic-stig-audit v1.0

Elasticsearch security benchmark — xpack security, TLS transport and REST, authentication, authorization, audit logging, and container runtime checks. Includes CVE/KEV vulnerability scanning.

95 tests CIS ES CVE/KEV
pg-stig-audit v1.0

PostgreSQL CIS Benchmark v1.1 and DISA STIG. Checks pg_hba.conf, SSL, logging, privilege separation, and extension controls. FedRAMP High and CMMC Level 2 mapped.

CIS PG16 DISA STIG FedRAMP High
kafka-stig-audit v1.0

Kafka container security audit. SASL authentication, TLS inter-broker encryption, topic authorization, network exposure, and Zookeeper hardening.

67 tests CIS Kafka NIST 800-53
mongo-stig-audit v1.0

MongoDB CIS/STIG-style audit covering authentication, role-based access, encryption at rest, TLS, audit logging, and container runtime security.

67 tests CIS MongoDB MITRE ATT&CK
mark-sentinel (AI) Coming soon

Security audit for AI runtimes. Custom STIG-style controls for model I/O safety, agent behavior, supply chain integrity, runtime monitoring, and governance.

Framework coverage

Every finding is already mapped.

No manual cross-referencing. Each check carries its control IDs so you can filter, sort, and report by framework without additional work.

CIS Benchmarks

Redis, Elasticsearch, PostgreSQL 16, Kafka, MongoDB. Section and control IDs on every finding.

NIST 800-53 & 800-171

Rev 5 control IDs for every check. Suitable for FedRAMP package preparation and FISMA reporting.

FedRAMP & DISA STIG

FedRAMP Moderate and High mappings. DISA STIG IDs for PostgreSQL. Pre-formatted for eMASS import.

CMMC Level 2

Practice IDs and domain labels mapped to findings. Supports DoD contractor compliance documentation.

MITRE ATT&CK & D3FEND

Technique IDs on relevant findings. Connects configuration gaps to known adversary techniques.

CVE / KEV

Elasticsearch audit includes live CVE/KEV scanning against the NVD and CISA Known Exploited Vulnerabilities catalog.
